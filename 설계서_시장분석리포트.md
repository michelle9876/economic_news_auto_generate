## 1. 작업 컨텍스트

### 배경

매일 직접 여러 사이트(Yahoo Finance, Bloomberg, 뉴스 등)를 방문하며 시장 데이터를 수집하고 분석 리포트를 작성하는 작업을 반복하고 있다. 이 작업은 패턴이 명확하고 시간이 많이 걸리는 반복 작업으로, AI 에이전트 자동화에 적합하다.

### 목적

하루 2회(아침·마감) 주식·거시경제·금리환율·부동산 시장 데이터를 자동 수집하고, 수치 정리와 인사이트 해설을 포함한 Markdown 리포트를 생성한다.

### 범위

- **포함**: 주가지수 수집, 거시경제 지표 수집, 경제 뉴스 수집, 국내 부동산·한은 데이터 수집, 인사이트 해설 작성, 품질 검증, Markdown 리포트 저장
- **제외**: 개별 종목 심층 분석(확장 계획), 알림 시스템 발송(확장 계획), 투자 추천 및 매매 실행

### 입력 정의

| 입력 | 형식 | 설명 | 예시 |
|------|------|------|------|
| 실행 시점 | string | 아침 또는 마감 구분 | `"morning"` 또는 `"closing"` |
| 실행 날짜 | string (YYYYMMDD) | 리포트 날짜 | `"20260408"` |

### 출력 정의

| 출력 | 형식 | 위치 | 설명 |
|------|------|------|------|
| 최종 리포트 | `.md` | `/output/final/` | 수치 + 뉴스 요약 + 인사이트 포함 |
| 수집 원시 데이터 | `.json` | `/output/raw/` | 단계 간 데이터 전달용 |
| 검증 로그 | `.json` | `/output/logs/` | 품질 검증 결과 기록 |

### 제약조건

- API 키: NewsAPI, FRED API 키 보유 (환경변수로 관리)
- 데이터 수집 실패 시: N/A 표시 후 부분 발행 (발행 중단 없음)
- 실행 주기: 매일 (주말·공휴일 포함)
- 산출물 형식: Markdown (.md) 고정

### 용어 정의

| 용어 | 정의 |
|------|------|
| 아침 리포트 | 전일 마감 데이터 기반, 당일 시장 예비 브리핑 |
| 마감 리포트 | 당일 마감 데이터 기반, 당일 시장 결산 정리 |
| 거시지표 | GDP, 물가, 고용, 금리 등 경제 전체를 나타내는 지표 |
| 인사이트 | 단순 수치 나열이 아닌, 데이터의 의미·맥락·시사점 해설 |
| N/A | 데이터 수집 실패 시 해당 항목에 표시하는 값 |

## 3. 워크플로우

### 전체 흐름도

```
[입력: 실행 시점 + 날짜]
        ↓
1. CEO: 실행 시작
        ↓ (병렬)
2a. SCV(collect_market_data.py): 주가지수 수집
2b. SCV(collect_macro_data.py): 거시지표·금리·환율 수집
2c. SCV(collect_news.py): 경제 뉴스 수집
        ↓ (병렬 완료 후)
3. 리서처: 국내 데이터 WebSearch + 전체 데이터 통합·요약
        ↓
4. 애널리스트: 인사이트·해설 작성 + 자체 검토
        ↓
5. SCV(validate_report.py): 품질 검증
        ↓
6. CEO: 최종 검토 후 승인
        ↓
[출력: /output/final/YYYYMMDD_[morning|closing]_report.md]
```

### 단계별 상세

#### 1단계: CEO - 실행 시작

- **처리 주체**: CEO (판단)
- **처리 내용**: 실행 시점 확인, 날짜 설정, 하위 단계 호출 계획
- **성공 기준**: 실행 파라미터 정상 인식
- **실패 시**: 사용자에게 파라미터 재입력 요청

#### 2a단계: SCV - 주가지수 수집

- **처리 주체**: SCV 스크립트 (collect_market_data.py)
- **처리 내용**: yfinance로 KOSPI(^KS11), KOSDAQ(^KQ11), S&P500(^GSPC), NASDAQ(^IXIC), 다우(^DJI), 닛케이(^N225) 수집
- **출력**: `/output/raw/market_data.json`
- **성공 기준**: 최소 3개 이상 지수 수집 성공
- **실패 시**: 수집된 지수만 저장, 나머지 N/A 처리 후 계속 진행

#### 2b단계: SCV - 거시지표·금리·환율 수집

- **처리 주체**: SCV 스크립트 (collect_macro_data.py)
- **처리 내용**: FRED API로 미국 10년물 금리(DGS10), 연방기금금리(FEDFUNDS), 원달러 환율(DEXKOUS), CPI(CPIAUCSL) 수집
- **출력**: `/output/raw/macro_data.json`
- **성공 기준**: 금리 또는 환율 중 최소 1개 수집 성공
- **실패 시**: N/A 처리 후 계속 진행

#### 2c단계: SCV - 경제 뉴스 수집

- **처리 주체**: SCV 스크립트 (collect_news.py)
- **처리 내용**: NewsAPI로 최근 24시간 경제 뉴스 수집 (키워드: economy, market, Fed, 한국경제)
- **출력**: `/output/raw/news_data.json`
- **성공 기준**: 최소 3개 뉴스 수집
- **실패 시**: 수집된 뉴스만 포함, 3개 미만이면 검증 단계에서 경고 플래그

#### 3단계: 리서처 - 데이터 통합·요약

- **처리 주체**: 리서처 직원 (LLM 판단)
- **처리 내용**:
  1. WebSearch로 국내 부동산 시세, 한국은행 기준금리 최신 데이터 수집
  2. 2a~2c 수집 결과 검토 및 N/A 항목 확인
  3. 전체 데이터 통합, 리포트 데이터 섹션 초안 작성
- **출력**: `/output/processed/data_summary.json`
- **성공 기준**: 4개 영역(주식·거시·금리환율·부동산) 중 최소 2개 이상 데이터 존재
- **검증 방법**: LLM 자기 검증 (누락 항목 스스로 확인)
- **실패 시**: CEO에게 에스컬레이션 (부동산 데이터 누락 시 스킵 + 로그)

#### 4단계: 애널리스트 - 인사이트 작성

- **처리 주체**: 애널리스트 직원 (LLM 판단)
- **처리 내용**:
  1. data_summary.json 기반 시장 해석·인사이트 작성
  2. 주요 수치 변화의 의미, 투자자 관점 시사점, 주목할 이벤트 포함
  3. 완성 후 자체 검토 (인사이트 완성도, 톤, 길이 점검)
- **출력**: `/output/drafts/insight_draft.md`
- **성공 기준**: 인사이트 섹션 200자 이상, 단순 수치 나열이 아닌 해석 포함
- **검증 방법**: LLM 자기 검증 (자체 완성도 판단 후 CEO에 보고)
- **실패 시**: 자동 재작업 최대 2회 → 초과 시 CEO에 에스컬레이션

#### 5단계: SCV - 품질 검증

- **처리 주체**: SCV 스크립트 (validate_report.py)
- **처리 내용**:
  1. 필수 수치 존재 여부 확인 (주가지수 1개 이상, 금리 또는 환율 1개 이상)
  2. 뉴스 항목 수 확인 (3개 이상)
  3. 인사이트 섹션 길이 확인 (200자 이상)
- **출력**: `/output/logs/validation_YYYYMMDD_[morning|closing].json`
- **성공 기준**: 3개 항목 모두 통과
- **실패 시**: 경고 플래그 포함하여 CEO에 결과 전달 (발행 중단 없음)

#### 6단계: CEO - 최종 검토·승인

- **처리 주체**: CEO (LLM 판단)
- **처리 내용**:
  1. 검증 결과 확인
  2. 인사이트 초안 + 데이터 요약 합쳐 최종 리포트 생성
  3. N/A 항목이 있는 경우 리포트 상단에 데이터 수집 제한 안내 추가
  4. `/output/final/`에 저장
- **출력**: `/output/final/YYYYMMDD_[morning|closing]_report.md`
- **성공 기준**: 파일 저장 완료
- **실패 시**: 로그 기록 후 사용자에게 오류 보고

## 4. 구현 스펙

### 폴더 구조

```
/01_경제관련/

│       ├── collect_market_data.py         # yfinance 주가지수 수집
│       ├── collect_macro_data.py          # FRED API 거시지표 수집
│       ├── collect_news.py                # NewsAPI 뉴스 수집

└── /output/
    ├── /raw/                              # SCV 수집 원시 데이터

    ├── /drafts/                           # 애널리스트 초안
    ├── /logs/                             # 검증 로그

```

### CEO(CLAUDE.md) 핵심 섹션 목록

| 섹션 | 내용 |
|------|------|
| 프로젝트 개요 | 일일 시장 분석 리포트 2회 생성, 범위 4개 영역 |
| 용어 정의 | 아침/마감 리포트, N/A 처리 정책 |
| 조직도 | 리서처·애널리스트 파일 경로, 보고 체계 |
| 워크플로우 | 6단계 흐름, 병렬 수집(2a/2b/2c) 명시 |
| 호출 규칙 | 직원 호출 방법 (`/.claude/staff/experts/researcher.md` 읽기 후 역할 수행) |
| 품질 게이트 | 검증 통과 기준, N/A 허용 정책 |
| 실패 처리 | N/A 표시 후 부분 발행, 에스컬레이션 조건 |

### 직원별 파일 및 핵심 섹션

#### researcher.md 핵심 섹션

- 역할 정의: 데이터 수집 에이전트, SCV 결과 검토자
- WebSearch 수집 대상: 한국은행 기준금리, KB부동산 시세, 국토부 실거래가
- 데이터 통합 규칙: N/A 처리 방식, data_summary.json 스키마
- 자기 검증 체크리스트: 4개 영역 최소 데이터 존재 여부

#### analyst.md 핵심 섹션

- 역할 정의: 시장 해설가, 인사이트 작성자
- 인사이트 작성 가이드: 수치 → 의미 → 시사점 구조
- 품질 기준: 200자 이상, 수치와 해석 혼합, 투자자 관점
- 자기 검증 체크리스트: 완성도, 길이, 톤 점검 후 CEO 보고

### SCV 스크립트 목록 및 입출력 스키마

#### collect_market_data.py

```
입력: 없음 (실행일 자동 감지)
처리: yfinance로 ^KS11, ^KQ11, ^GSPC, ^IXIC, ^DJI, ^N225 수집
출력: /output/raw/market_data.json
스키마:
{
  "collected_at": "2026-04-08T06:00:00",
  "indices": {
    "KOSPI": {"value": 2540.1, "change_pct": 0.3, "status": "ok"},
    "SP500": {"value": null, "change_pct": null, "status": "failed"}
  }
}
```

#### collect_macro_data.py

```
입력: 환경변수 FRED_API_KEY
처리: FRED API로 DGS10, FEDFUNDS, DEXKOUS, CPIAUCSL 수집
출력: /output/raw/macro_data.json
스키마:
{
  "collected_at": "2026-04-08T06:00:00",
  "indicators": {
    "us_10y_yield": {"value": 4.21, "unit": "%", "status": "ok"},
    "usd_krw": {"value": 1320.5, "unit": "KRW", "status": "ok"}
  }
}
```

#### collect_news.py

```
입력: 환경변수 NEWS_API_KEY
처리: NewsAPI로 최근 24시간 경제 뉴스 수집 (최대 10개)
      키워드: "economy OR market OR Fed OR 한국경제 OR 코스피"
출력: /output/raw/news_data.json
스키마:
{
  "collected_at": "2026-04-08T06:00:00",
  "count": 7,
  "articles": [
    {"title": "...", "source": "...", "published_at": "...", "url": "..."}
  ]
}
```

#### validate_report.py

```
입력: /output/raw/market_data.json, /output/raw/macro_data.json,
      /output/raw/news_data.json, /output/drafts/insight_draft.md
처리:
  - 필수 수치: 주가지수 ok 1개 이상 + (금리 or 환율) ok 1개 이상
  - 뉴스: count >= 3
  - 인사이트: 문자 수 >= 200
출력: /output/logs/validation_YYYYMMDD_[morning|closing].json
스키마:
{
  "validated_at": "...",
```
